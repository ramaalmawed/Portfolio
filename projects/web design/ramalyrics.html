RamaLyrics Web App

RamaLyrics is a React-based music and lyrics web application where users can search for songs, artists, and albums, read lyrics, and listen to music through Spotify.

Project Overview

The goal of RamaLyrics was to create a clean, modern, and user-friendly music platform that combines song search, lyrics, artist information, album covers, and Spotify playback in one simple interface.

Project Type

React web app · Front-end development · API integration · Responsive design

Main Goal

To build an easy-to-use platform where users can discover music and read lyrics.

My Role

UI design, front-end development, API integration, responsive layout, testing and troubleshooting.

APIs

Spotify API for music data and Lyrics.ovh API for lyrics.

Concept & Idea

A Simple Music Discovery Platform

The idea behind RamaLyrics was to create a simple platform for discovering music, searching for songs and artists, and reading lyrics without making the experience feel complicated.

I wanted the app to feel clean and easy to understand, with a design that helps users focus on the song, the artist, the album cover, and the lyrics.

Some early ideas included searching artists by first letter, saving favorite songs, and using a heart button for favorites. These ideas were planned, but not all of them were included because of the project time limit.

Main Features

Song Search

Users can search for songs, artists, and albums through Spotify data.

Lyrics View

Lyrics are fetched through Lyrics.ovh when available.

Spotify Player

Spotify iframe was used so users can listen directly through Spotify.

Responsive Layout

The app was designed to work on desktop, tablet, and mobile screens.

Design Direction

Clean, Readable & Responsive

The design focused on readability, especially because lyrics are long text content. I used Lora for lyric text because it gives a softer reading experience, and Josefin Sans for headings and navigation to create a more modern style.

I used Flexbox and media queries to make the layout responsive. Search results appear in multiple columns on larger screens and become simpler on mobile screens.

Small hover effects and animations were added to make clickable elements feel more interactive and easier to understand.

Design: Responsive Layout · Typography · Hover Effects · UI Structure

Development Process

01

React Structure

I built the app with React and divided the interface into reusable components, such as search, lyrics, header, footer, and result sections.

02

Spotify API

Spotify API was used to fetch songs, artists, album covers, track information, popularity, and other music-related data.

03

Lyrics.ovh API

Lyrics.ovh was used to fetch lyrics because it was simple to integrate and did not require authentication.

04

Spotify Embed

I used Spotify iframe to add the music player instead of building a custom audio player.

05

React Router

React Router was used to manage navigation and routes between different parts of the app.

06

Responsive Testing

I tested the layout on different screen sizes and adjusted the interface using media queries.

Challenges & Solutions

API Choice

I first explored Genius API, but it did not provide lyrics in the way I needed. I also considered Musixmatch, but API access was not available for students. I later used Lyrics.ovh for lyrics.

Spotify Authentication

Spotify API required an access token. I solved this by following Spotify’s documentation and using a Base64 encoded Client ID and Client Secret.

Special Characters

Some artist and song names included special characters. I used encodeURIComponent to make sure names were correctly formatted in API requests.

Routing After Build

React Router caused navigation issues after building the app. The solution was to use basename so routes worked correctly after deployment.
